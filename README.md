# 智慧酒店管理平台

## 项目简介

基于 **Spring Boot + MyBatis-Flex + Vue + JWT + MySQL** 的智慧酒店管理平台，在权限管理系统基础上扩展实现酒店基础架构管理、预订管理、入住退房管理、高级财务分析管理、会员等级管理系统，包括酒店信息、楼栋楼层、房型分类、客房管理、客户档案、预订管理、入住登记、换房续住、费用结算、发票管理、收款分析、坏账处理、综合财务分析、财务看板、会员等级体系、会员注册、积分管理、等级权益、会员统计等核心功能模块。打通从房源查询、预订创建、入住登记到退房结算、会员运营、财务分析的完整业务链路。

> [!NOTE]
> **技术说明**: 原始需求中的 "mybatis-flux" 为笔误，实际采用的是 **MyBatis-Flex** 技术栈。MyBatis-Flex 是一个优雅的 MyBatis 增强框架，提供了更简洁的 API 和更好的性能。

## 技术栈

### 后端
- **框架**: Spring Boot 2.7.x
- **ORM**: MyBatis-Flex 1.7.x
- **数据库**: MySQL 8.0
- **安全**: Spring Security + JWT
- **连接池**: HikariCP

### 前端
- **框架**: Vue 3
- **构建工具**: Vite
- **UI 组件**: Element Plus
- **路由**: Vue Router
- **状态管理**: Pinia

## 快速开始

> [!TIP]
> 使用 Docker Compose 一键启动，无需本地安装 Java、Node.js 等环境

### 环境变量配置

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `JWT_SECRET` | JWT 签名密钥（**必须设置**，至少32字符） | 无（占位值，启动时校验） |
| `CORS_ALLOWED_ORIGINS` | 允许的跨域来源（逗号分隔） | `http://localhost:3000` |

> [!IMPORTANT]
> 生产部署前 **必须** 通过环境变量设置 `JWT_SECRET`，否则后端将拒绝启动。
> 可在项目根目录创建 `.env` 文件供 Docker Compose 读取：
> ```env
> JWT_SECRET=YourSuperSecretKeyAtLeast32Characters!!
> CORS_ALLOWED_ORIGINS=https://your-domain.com
> ```

### Docker 部署（推荐）

```bash
# 一键启动
docker compose up -d --build

# 查看日志
docker compose logs -f
```

## 功能模块

### 系统管理模块
- ✅ **用户管理** - 用户增删改查、状态切换、密码重置
- ✅ **角色管理** - 角色配置、权限分配
- ✅ **菜单权限** - 菜单树维护、权限标识配置

### 酒店管理模块
- ✅ **酒店信息配置** - 酒店名称、品牌、星级、地址、联系方式、营业执照、开业时间、酒店简介
- ✅ **服务设施管理** - 停车场、游泳池、健身房等设施管理，记录开放时间
- ✅ **酒店图片管理** - 多图上传、设置主图、排序、删除
- ✅ **楼栋管理** - 楼栋增删改查、编号、总楼层数、启用/停用状态
- ✅ **楼层管理** - 楼层号/名称、楼层特点（无烟/安静等）、房间数量自动统计、楼层状态
- ✅ **房型管理** - 房型编码、面积、床型、入住人数、设施清单、图片、平日/周末/成本价格
- ✅ **房间管理** - 房间号唯一、楼栋/楼层/房型联动、朝向、景观、位置特点、特殊标识
- ✅ **房间状态管理** - 状态流转（空闲→已预订→已入住→待清洁→清洁中→空闲），变更记录
- ✅ **批量创建房间** - 选择楼栋/楼层/房型，输入房号前缀和起止号，预览确认创建
- ✅ **房间搜索筛选** - 房号搜索、楼栋/楼层联动筛选、房型/状态/朝向/景观多条件筛选
- ✅ **统计看板** - 总体数据、房型分布、房间状态统计、楼层分布图表

### 高级财务分析管理模块
- ✅ **发票管理** - 开票申请（关联入住单/预订单、普通发票/专用发票）、开票处理（发票代码号码、PDF扫描件、电子发票邮箱发送）、发票邮寄（收件人、快递信息）、发票查询统计（按号码/客户/关联单搜索、按类型/状态/时间筛选、开票总额笔数统计）
- ✅ **收款明细分析** - 收款明细报表（日期范围、收款类型/支付方式筛选、导出）、收款趋势分析（近30日趋势折线图、支付方式分类趋势、周对比）、支付方式偏好分析（占比饼图、金额柱状图、现金使用率趋势）、收款人员统计（人员排名、详细记录查看）
- ✅ **坏账处理** - 坏账识别（自动识别超期挂账/逃单/拖欠）、催收记录（电话/短信/上门）、部分收回、坏账核销（需财务经理或总经理审批）、法律追偿（字段预留）、坏账统计与趋势分析、风险预警
- ✅ **综合财务分析** - 营收分析（营收构成饼图、趋势图、房型分析、RevPAR/ADR/入住率指标）、成本利润分析（字段预留）、应收账款分析（账龄分析、协议单位分析、周转率、逾期统计）、现金流分析（流入流出明细、趋势图、健康度）
- ✅ **综合财务看板** - 财务总览看板（今日/本月数据、关键指标、趋势迷你图、预警提示）、对比分析（时间对比、目标对比、房型对比、渠道对比）

### 会员等级管理系统
- ✅ **等级体系配置** - 多等级管理（普通/银卡/金卡/钻石）、等级图标颜色、升级条件（消费金额/积分）、保级条件（年度消费）、等级排序、启用状态
- ✅ **等级权益配置** - 房费折扣、餐饮折扣（字段预留）、积分倍率、专属服务（免费升级房型/延迟退房/提前入住/免费早餐/免费接送机/专属客服/生日礼遇）、押金减免、其他权益（房间优选/积分兑换优先/预订优先）
- ✅ **会员注册** - 客户档案升级为会员、新客户直接注册、会员卡号自动生成（MB+年月日+6位序号）、初始等级默认最低、注册来源、推荐人（字段预留）
- ✅ **会员信息管理** - 基础信息（继承客户档案）、会员专属信息（卡号/等级/注册时间/累计积分/当前积分/累计消费/本年度消费/入住次数/上次入住）、推荐人信息
- ✅ **会员列表查询** - 列表字段（卡号/姓名/手机号/等级/积分/消费/注册时间/上次入住/状态）、多条件筛选（关键词/等级/注册时间/积分范围/状态）、排序方式、批量操作（批量发放积分/批量导出）
- ✅ **会员详情** - 顶部会员信息展示、操作按钮（发放积分/调整等级/冻结解冻）、信息卡片（基础信息/会员信息/等级权益/积分明细/等级变更记录）、入住历史（字段预留）
- ✅ **积分管理** - 手动发放积分（活动赠送/客诉补偿/生日礼遇/推荐奖励/其他）、积分使用（字段预留）、积分明细记录（时间轴/表格）、积分日志不可删除
- ✅ **等级调整** - 手动调整等级、调整原因记录、等级变更日志（升级/降级/手动调整）
- ✅ **会员冻结解冻** - 冻结后无法享受权益和使用积分、冻结原因记录、操作日志
- ✅ **会员统计** - 总体数据（会员总数/本月新增/本月活跃）、等级分布饼图、近12个月增长趋势折线图
- ✅ **权限控制** - 酒店管理员（全部权限）、前厅部经理（查看+发放积分+调整等级）、前台员工（注册+查看）、客服人员（查看+发放积分）、财务人员（查看消费数据）

## 使用指南

### 登录测试

### 酒店管理操作流程

1. **配置酒店信息**：使用 hotel_admin 登录 → 酒店管理 → 酒店概览 → 填写酒店基本信息
2. **创建楼栋楼层**：酒店管理 → 楼栋楼层 → 新增楼栋 → 在楼栋下新增楼层
3. **创建房型**：酒店管理 → 房型管理 → 新增房型，填写面积、床型、设施、价格等
4. **创建房间**：酒店管理 → 房间管理 → 新增房间（单个）或批量创建
5. **管理房间状态**：房间管理 → 修改状态，或进入房间详情页操作
6. **查看统计**：酒店管理 → 统计看板

### 权限验证示例

- 使用 **receptionist** 登录：只能看到房间管理菜单，房间列表只显示空闲和已预订的房间，看不到成本价
- 使用 **finance_staff** 登录：可以看到酒店概览、房型管理、房间管理、统计看板，能看到成本价和导出按钮，但不能修改任何数据
- 使用 **frontdesk_manager** 登录：可以查看所有信息，修改房间属性和状态，但看不到编辑价格和删除按钮
- 使用 **finance_manager** 登录：可以看到所有财务分析页面，审批坏账核销，查看成本利润分析
- 使用 **hotel_admin** 登录：所有财务权限，审批坏账核销，查看所有分析

### 高级财务分析使用指南

1. **发票管理**：财务管理 → 发票管理 → 申请开票 → 填写发票信息 → 财务人员处理开票 → 邮寄发票
2. **收款分析**：财务管理 → 收款明细报表/收款趋势分析/支付方式分析/收款人员统计 → 选择日期范围 → 查看分析

4. **营收分析**：财务管理 → 营收分析 → 选择日期范围 → 查看营收构成、趋势、房型分析
5. **应收账款分析**：财务管理 → 应收账款分析 → 查看账龄分布、协议单位风险
6. **现金流分析**：财务管理 → 现金流分析 → 查看流入流出明细和趋势
7. **财务总览看板**：财务管理 → 财务总览看板 → 查看今日/本月关键数据和预警
8. **对比分析**：财务管理 → 对比分析 → 选择对比维度 → 查看增长率和对比图表

### 会员管理使用指南

1. **等级配置**：使用 admin 或 hotel_admin 登录 → 会员管理 → 会员等级管理 → 查看/新增/编辑/删除等级 → 配置升级条件和权益
2. **会员注册（已有客户）**：会员管理 → 会员列表 → 点击"注册会员" → 选择"已有客户升级" → 搜索并选择客户 → 填写注册来源和推荐人 → 提交

5. **调整等级**：会员列表 → 点击"调整等级" 或进入会员详情页 → 选择新等级 → 填写调整原因 → 确认调整
6. **冻结/解冻会员**：会员列表 → 点击"冻结"或"解冻" → 填写原因 → 确认
7. **查看会员详情**：会员列表 → 点击"详情" → 查看基础信息、会员信息、等级权益、积分明细、等级变更记录
8. **会员统计**：会员管理 → 会员统计 → 查看总体数据、等级分布饼图、增长趋势折线图
9. **批量发放积分**：会员列表 → 勾选多个会员 → 点击"批量发放积分" → 输入积分数量 → 确认

## 项目结构

```
├── backend/          # 后端 Spring Boot 项目
│   ├── src/main/java/com/example/permission/
│   │   ├── config/      # 配置类
│   │   ├── controller/  # 控制器（系统+酒店+入住）
│   │   ├── entity/      # 实体类
│   │   ├── mapper/      # MyBatis-Flex Mapper
│   │   ├── service/     # 服务层
│   │   ├── security/    # JWT 安全配置
│   │   └── common/      # 公共类
│   └── pom.xml
├── frontend/         # 前端 Vue 3 项目
│   ├── src/
│   │   ├── views/       # 页面组件
│   │   │   ├── system/      # 系统管理页面
│   │   │   ├── hotel/       # 酒店管理页面
│   │   │   ├── booking/     # 预订管理页面
│   │   │   ├── checkin/     # 入住退房页面
│   │   │   ├── finance/     # 财务管理页面
│   │   │   ├── member/      # 会员管理页面
│   │   │   └── maintenance/ # 维护管理页面
│   │   ├── router/      # 路由配置
│   │   ├── stores/      # 状态管理
│   │   └── api/         # API 接口
│   └── package.json
├── db/               # 数据库脚本
│   ├── init.sql      # 初始化脚本（含酒店表和权限数据）
│   ├── migration_20260615_customer.sql  # 客户管理系统
│   ├── migration_20260616_booking_system.sql  # 预订管理系统
│   ├── migration_20260617_checkin_system.sql  # 入住退房管理系统
│   ├── migration_20260618_finance.sql         # 基础财务结算管理
│   ├── migration_20260618_finance_advanced.sql # 高级财务分析管理
│   └── migration_20260618_member_system.sql    # 会员等级管理系统
└── docker-compose.yml
```

## Docker 配置

| 容器名 | 服务 |
|--------|------|
| label-530-db | MySQL 数据库 |
| label-530-backend | Spring Boot 后端 |
| label-530-frontend | Vue 3 前端 |

## 常用命令

```bash
# 启动所有服务
docker compose up -d --build

# 查看后端日志
docker compose logs -f backend

# 进入数据库
docker exec -it label-530-db mysql -uroot -proot permission_system

# 停止所有服务
docker compose down
```

## 本地开发

### 环境要求
- JDK 17+
- Node.js 18+
- MySQL 8.0+

### 后端启动
```bash
cd backend
mvn spring-boot:run
```

### 前端启动
```bash
cd frontend
npm install
npm run dev
```
