# 智慧酒店管理平台

## 项目简介

基于 **Spring Boot + MyBatis-Flex + Vue + JWT + MySQL** 的智慧酒店管理平台，在权限管理系统基础上扩展实现酒店基础架构管理、预订管理、入住退房管理、高级财务分析管理、会员等级管理系统、积分规则与应用系统、会员权益自动应用系统、会员等级升降级机制、会员数据深度分析系统、客户评价反馈体系、评价数据统计分析、投诉处理系统，包括酒店信息、楼栋楼层、房型分类、客房管理、客户档案、预订管理、入住登记、换房续住、费用结算、发票管理、收款分析、坏账处理、综合财务分析、财务看板、会员等级体系、会员注册、积分管理、等级权益、会员统计、积分规则配置、积分自动获取、积分抵扣兑换、积分明细查询、积分统计分析、预订权益自动应用、入住权益自动应用、退房权益自动应用、权益使用记录查询、自动升级、保级降级、等级变更记录、升降级统计、定时任务管理、会员价值分析、等级分布分析、行为特征分析、流失预警、权益成本统计、评价指标配置、评价标签配置、快捷评语管理、评价配置预览、评价邀请管理、评价链接生成、H5评价页面、评价提交处理、会员评价积分奖励、评价审核管理、评价公开展示、评价回复管理、评价总览分析、评价趋势分析、评价指标雷达图、指标评分对比、指标改进趋势、高频标签统计、标签使用趋势、房型评价对比、房型指标分析、投诉提交（H5）、投诉受理分配、投诉处理跟进、投诉回访记录等核心功能模块。打通从房源查询、预订创建、入住登记到退房结算、会员运营、积分运营、财务分析、会员数据分析、客户评价反馈、评价数据分析、评价审核展示、投诉处理的完整业务链路。

> [!NOTE]
> **技术说明**: 原始需求中的 "mybatis-flux" 为笔误，实际采用的是 **MyBatis-Flex** 技术栈。MyBatis-Flex 是一个优雅的 MyBatis 增强框架，提供了更简洁的 API 和更好的性能。

## 技术栈

### 后端
- **框架**: Spring Boot 2.7.x
- **ORM**: MyBatis-Flex 1.7.x
- **数据库**: MySQL 8.0
- **安全**: Spring Security + JWT
- **连接池**: HikariCP

### 前端
- **框架**: Vue 3
- **构建工具**: Vite
- **UI 组件**: Element Plus
- **路由**: Vue Router
- **状态管理**: Pinia

## 快速开始

> [!TIP]
> 使用 Docker Compose 一键启动，无需本地安装 Java、Node.js 等环境

### 环境变量配置

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `JWT_SECRET` | JWT 签名密钥（**必须设置**，至少32字符） | 无（占位值，启动时校验） |
| `CORS_ALLOWED_ORIGINS` | 允许的跨域来源（逗号分隔） | `http://localhost:3000` |

> [!IMPORTANT]
> 生产部署前 **必须** 通过环境变量设置 `JWT_SECRET`，否则后端将拒绝启动。
> 可在项目根目录创建 `.env` 文件供 Docker Compose 读取：
> ```env
> JWT_SECRET=YourSuperSecretKeyAtLeast32Characters!!
> CORS_ALLOWED_ORIGINS=https://your-domain.com
> ```

### Docker 部署（推荐）

```bash
# 一键启动
docker compose up -d --build

# 查看日志
docker compose logs -f
```

- ✅ **自动生成评价邀请** - 客人退房完成后系统自动触发，创建评价邀请记录（关联入住单号、客人信息、房型、入住/退房日期），生成唯一评价链接（8位随机字母数字验证码），设置7天有效期，评价状态为"待评价"
- ✅ **评价邀请列表** - 表格显示所有邀请记录（入住单号、客人姓名、手机号、房型、入住日期、邀请时间、评价状态、是否发送、发送时间），支持按客人姓名/手机号/入住单号搜索、按评价状态筛选、按日期范围筛选，按邀请时间倒序
- ✅ **复制评价链接** - 点击复制完整评价链接到剪贴板（`http://localhost:8080/h5/review?order=入住单号&code=验证码`），提示"链接已复制，可用于测试"
- ✅ **模拟发送评价邀请** - 弹出对话框选择发送方式（短信/邮件），确认后更新发送状态为"已发送"、记录发送时间和发送方式，不实际发送仅模拟标记状态
- ✅ **手动创建评价邀请** - 入住单详情页增加"邀请评价"按钮，判断是否已退房、是否已有邀请，创建成功后显示评价链接可复制
- ✅ **客人评价页面（H5）** - 酒店信息展示、入住信息卡片、综合评分显示（大字、实时计算）、评价指标星级评分（1-5星、必评指标红星标记）、评价标签选择（卡片式多选、好评绿色/差评红色）、评价内容填写（500字限制、字数统计、快捷评语按钮）、图片上传（最多6张、JPG/PNG格式、单张5MB限制、预览删除）、匿名评价复选框、固定底部提交按钮
- ✅ **评价链接访问验证** - 验证入住单号和验证码匹配、验证链接是否过期（7天有效期）、验证是否已评价，验证失败显示"链接已失效"或"您已评价过该订单"
- ✅ **评价提交处理** - 保存评价记录（各指标评分、综合评分、标签、内容、图片、是否匿名）、更新评价邀请状态为"已评价"、会员评价自动赠送积分（根据积分规则配置）、评价状态为"待审核"
- ✅ **提交成功页面** - 显示感谢文案、会员积分赠送提示（"已赠送XX积分到您的账户"）、优惠券领取入口（字段预留）、返回首页按钮（字段预留）
- ✅ **综合评分计算** - 根据各指标评分和权重自动计算：综合评分 = Σ(指标评分 × 指标权重)，前端实时更新显示
- ✅ **权限控制** - 酒店管理员（查看所有评价邀请、手动创建邀请、复制链接、模拟发送）、前厅部经理（查看评价邀请、手动创建邀请、复制链接）、客服人员（查看评价邀请、复制链接、模拟发送）、前台员工（退房时触发自动邀请、查看邀请记录）、客人（无需登录，通过评价链接访问H5页面提交评价）

### 积分规则与应用系统
- ✅ **积分获取规则配置** - 支持6种积分获取类型（消费积分/签到积分/评价积分/推荐积分/生日积分/活动积分），每种类型可配置不同参数（消费积分：每消费X元获得Y积分；签到积分：每次签到获得X积分等），支持积分倍率（根据会员等级调整），规则生效时间范围，启用/禁用状态
- ✅ **积分消耗规则配置** - 支持2种积分消耗类型（积分抵现/积分兑换），积分抵现规则（兑换比例、单次最多使用积分、最低消费要求、抵扣上限百分比），积分兑换规则（商品名称、所需积分、库存，字段预留），适用范围（会员等级/生效时间），启用/禁用状态
- ✅ **消费积分自动获取** - 入住退房时自动计算并发放积分，计算公式：实际获得积分 = 基础积分 × 等级倍率，积分发放提示（消费金额、基础积分、等级倍率、实际获得积分、更新后积分余额）
- ✅ **积分抵扣** - 会员预订或入住时使用积分抵扣房费，显示当前可用积分，积分使用输入或滑动条选择，自动计算抵扣金额，条件校验（积分数量、最低消费、抵扣上限），积分使用日志记录
- ✅ **积分兑换** - 字段预留，后续商城系统实现（免费升级券/免费早餐券/延迟退房券/优惠券）
- ✅ **积分明细查询** - 会员积分统计（当前积分/累计获得/累计使用），积分明细列表（时间/类型/方式/积分变动/关联订单/余额），按时间范围和类型筛选，管理员查看任意会员积分明细，导出功能
- ✅ **积分统计分析** - 积分总览（积分总量/本月发放/本月使用/使用率/积分负债），积分来源分析饼图（消费/活动/推荐/生日占比），积分去向分析饼图（抵现/兑换占比），近12个月积分趋势折线图，积分转化分析（活跃会员数/活跃率/平均使用频率/平均单次使用积分）
- ✅ **权限控制** - 酒店管理员（全部权限含规则配置）、前厅部经理（查看规则+查看统计）、前台员工（积分抵扣操作）、客服人员（查看积分明细+协助使用）、财务人员（查看积分统计/负债数据）

### 会员权益自动应用系统
- ✅ **预订场景权益应用** - 会员身份自动识别（选择房间后自动查询会员信息）、会员信息卡片展示（会员等级带图标、当前可用积分、权益概览）、房费折扣自动应用（显示原价划线价、折后价醒目标注、折扣说明如"金卡会员9折优惠"、折扣金额自动从订单总额扣除、订单详情记录折扣信息）、专属服务提示（延迟退房、免费升级、免费早餐等服务提示卡片）
- ✅ **入住场景权益应用** - 押金减免处理（显示标准押金金额、减免金额、实际需支付押金、减免说明如"金卡会员免押金"、减免后的押金金额应用到入住单、记录押金减免日志）、免费升级房型（系统提示"该会员可免费升级，是否升级？"、显示可升级房型列表更高等级且有空房、前台选择升级房型后分配新房型、订单房费不变仍按原房型价格、记录升级信息原房型新房型升级原因）、房间优选推荐（在可用房间列表中标记"会员优选"房间、优选条件景观好楼层高朝向佳离电梯近等、前台可选择推荐房间或其他可用房间）
- ✅ **退房场景权益应用** - 延迟退房处理（检查延迟退房权益、享有权益允许延迟至规定时间如下午2点普通客户需12点、延迟时间内不额外收费、系统记录延迟退房信息、不享有权益按标准退房时间执行超时需收取费用）、积分自动发放（退房结算时自动计算基础积分消费金额对应积分、应用等级倍率根据会员等级倍率调整如金卡1.5倍、实际获得积分=基础积分×等级倍率、积分自动发放到会员账户、显示积分发放提示本次消费获得XX积分当前积分余额XX分、记录积分发放日志）
- ✅ **权益使用记录** - 权益应用日志记录（会员卡号姓名、权益类型折扣升级押金减免延迟退房积分发放、关联订单号或入住单号、权益内容具体折扣金额升级房型减免金额等、应用时间、操作人前台员工）、会员可查看权益使用记录（在会员详情页增加"权益使用记录"标签页、显示该会员的权益使用历史使用时间权益类型权益内容关联订单、支持按时间范围筛选、支持导出记录）
- ✅ **权益使用记录列表** - 管理员可查看所有会员权益使用记录、支持按会员信息权益类型关联单号时间范围筛选、支持查看详情、支持导出记录
- ✅ **权限控制** - 酒店管理员（查看所有会员权益应用情况、可手动调整权益应用）、前厅部经理（查看会员权益应用记录、可处理特殊权益申请）、前台员工（在预订和入住时自动应用权益、可选择是否升级房型）、客服人员（查看会员权益使用记录、协助会员了解权益）

### 酒店管理操作流程

### 权限验证示例

### 会员管理使用指南

1. **等级配置**：使用 admin 或 hotel_admin 登录 → 会员管理 → 会员等级管理 → 查看/新增/编辑/删除等级 → 配置升级条件和权益
2. **会员注册（已有客户）**：会员管理 → 会员列表 → 点击"注册会员" → 选择"已有客户升级" → 搜索并选择客户 → 填写注册来源和推荐人 → 提交
3. **会员注册（新客户）**：会员管理 → 会员列表 → 点击"注册会员" → 选择"新客户注册" → 填写客户基础信息 → 提交
4. **发放积分**：会员列表 → 点击"发放积分" 或进入会员详情页 → 填写积分数量、发放原因和说明 → 确认发放
5. **调整等级**：会员列表 → 点击"调整等级" 或进入会员详情页 → 选择新等级 → 填写调整原因 → 确认调整
6. **冻结/解冻会员**：会员列表 → 点击"冻结"或"解冻" → 填写原因 → 确认
7. **查看会员详情**：会员列表 → 点击"详情" → 查看基础信息、会员信息、等级权益、积分明细、等级变更记录
8. **会员统计**：会员管理 → 会员统计 → 查看总体数据、等级分布饼图、增长趋势折线图
9. **批量发放积分**：会员列表 → 勾选多个会员 → 点击"批量发放积分" → 输入积分数量 → 确认

### 积分规则与应用使用指南

1. **配置积分获取规则**：使用 admin 或 hotel_admin 登录 → 会员管理 → 积分规则管理 → 积分获取规则标签页 → 点击"新增规则" → 选择规则类型（消费积分/签到积分/评价积分/推荐积分/生日积分/活动积分） → 根据类型填写规则参数（如消费积分：每消费X元获得Y积分） → 设置积分倍率、生效时间 → 启用规则
2. **配置积分消耗规则**：会员管理 → 积分规则管理 → 积分消耗规则标签页 → 点击"新增规则" → 选择规则类型（积分抵现/积分兑换） → 填写规则参数（如积分抵现：兑换比例100积分抵10元、单次最多使用积分、最低消费要求、抵扣上限百分比） → 选择适用会员等级和生效时间 → 启用规则
3. **退房自动获取积分**：办理退房时，系统自动根据消费积分规则计算积分（基础积分 × 等级倍率），退房成功后显示积分发放卡片，展示消费金额、基础积分、等级倍率、实际获得积分、更新后积分余额
4. **积分抵扣操作**：进入会员详情页 → 点击"积分抵扣"按钮 → 输入订单金额 → 使用滑动条选择使用积分数量 → 系统自动计算抵扣金额 → 确认抵扣
5. **查看积分明细**：会员管理 → 积分明细 → 选择会员 → 查看当前积分/累计获得/累计使用 → 按时间和类型筛选积分记录
6. **积分统计分析**：会员管理 → 积分统计 → 查看积分总览数据（积分总量/本月发放/本月使用/使用率/积分负债） → 查看积分来源/去向分析饼图 → 查看积分趋势折线图 → 查看积分转化分析

### 会员权益自动应用使用指南

#### 预订场景
1. **创建预订**：预订管理 → 新增预订 → 填写客户信息（如为会员系统自动识别）→ 选择房型和房间 → 系统自动识别会员身份并展示会员权益卡片
2. **查看会员权益**：预订页面右侧显示会员信息卡片，包含会员等级（带图标）、当前可用积分、可享受的权益概览
3. **房费折扣自动应用**：系统自动根据会员等级计算折扣，显示原价（划线价格）、折后价（醒目标注）、折扣说明（如"金卡会员9折优惠"），折扣金额自动从订单总额中扣除
4. **查看专属服务提示**：显示会员可享受的专属服务卡片，包括延迟退房、免费升级、免费早餐等权益提示
5. **提交预订**：确认订单信息后提交，系统自动记录折扣信息和权益应用日志

#### 入住场景
1. **办理入住**：入住管理 → 新增入住 → 选择预订单或散客入住 → 如为会员系统自动识别会员身份
2. **押金减免处理**：系统自动计算押金减免，显示标准押金金额、减免金额、实际需支付押金，以及减免说明（如"金卡会员免押金"或"银卡会员减免50%押金"）
3. **免费升级房型**：如会员享有升级权益且有空房，系统提示"该会员可免费升级，是否升级？"，显示可升级的房型列表（仅显示更高等级且有空房的房型），前台选择升级房型后系统自动分配新房型，订单房费不变（仍按原房型价格），并记录升级信息
4. **房间优选推荐**：在可用房间列表中，"会员优选"房间会被特殊标记（橙色边框、优选标签），优选条件包括景观好、楼层高、朝向佳、离电梯近等，前台可选择推荐房间或其他可用房间
5. **确认入住**：确认入住信息后提交，系统自动记录押金减免和房型升级信息

#### 退房场景

4. **查看积分发放提示**：退房成功后显示积分发放提示，包括"本次消费获得XX积分"、"当前积分余额：XX分"，并展示积分计算明细（消费金额、基础积分、等级倍率、实际获得积分）
5. **确认退房**：系统自动记录延迟退房和积分发放信息

### 会员等级升降级使用指南

#### 自动升级机制
1. **升级条件检查**：系统每日凌晨01:00自动执行升级检查任务，遍历所有正常状态的会员
2. **升级判断逻辑**：
   - 根据等级配置中的`upgradeType`判断升级条件类型（1=累计消费金额，2=累计积分）
   - 读取会员的累计消费金额或累计积分
   - 对比所有高于当前等级的已启用等级，找到符合条件的最高等级（支持跨级升级）
3. **升级执行**：
   - 更新会员等级为符合条件的最高等级
   - 记录等级变更日志：变更时间、原等级、新等级、变更类型（升级）、原因（达到升级条件）、触发方式（系统自动）
   - 升级后新订单立即享受新等级权益
4. **升级通知**：字段预留，后续对接短信/邮件/站内消息系统

#### 保级与降级机制
1. **保级检查时间**：系统每年1月1日凌晨01:00自动执行保级降级检查任务
2. **降级判断逻辑**：
   - 遍历所有非最低等级（普通会员）的正常会员
   - 读取会员的本年度消费金额（yearlySpent）
   - 对比当前等级的保级条件（keepCondition，年度消费金额要求）
   - 本年度消费金额未达到保级要求的，自动降级一级
3. **降级执行**：
   - 会员等级从当前等级降级到下一级（最低降至普通会员）
   - 记录等级变更日志：变更时间、原等级、新等级、变更类型（降级）、原因（未达保级条件）、触发方式（系统自动）
   - 降级后按新等级权益执行
4. **年度消费重置**：
   - 每年1月1日执行保级检查后，自动重置所有会员的`yearlySpent`（本年度消费金额）为0
   - `totalSpent`（累计消费金额）和`totalPoints`（累计积分）持续累加，不重置

#### 升降级记录管理
1. **等级变更日志**：每次等级变更（自动升级、自动降级、手动调整）均记录以下信息：
   - 变更时间（createTime）
   - 会员卡号（memberNo）、会员姓名（memberName）
   - 原等级（oldLevelName）、新等级（newLevelName）
   - 变更类型（changeType）：1=升级，2=降级，3=手动调整
   - 变更原因（reason）：达到升级条件、未达保级条件、手动调整原因
   - 触发方式（triggerType）：1=系统自动，2=管理员手动
   - 操作人（operatorId / operatorName）：手动调整时记录
2. **会员查看升降级历史**：
   - 会员详情页 → 等级变更记录标签页
   - 按时间倒序排列，时间轴或表格展示
   - 显示变更时间、原等级→新等级（带箭头）、变更类型和原因、触发方式
3. **管理员全局查询**：
   - 会员管理 → 等级变更记录
   - 支持按会员卡号、变更类型、触发方式、时间范围筛选

#### 升降级统计
1. **本月数据统计卡片**：
   - 本月升级人数
   - 本月降级人数
   - 本月净变动（升级人数 - 降级人数）
2. **按等级统计表**：
   - 升级到此等级人数

   - 带等级颜色徽章展示
3. **升降级趋势图**：

   - 双折线图 + 渐变填充面积
   - ECharts图表展示

#### 定时任务管理
1. **任务列表**：
   - 升级检查：每日凌晨01:00执行（Cron: `0 0 1 * * ?`）
   - 保级降级检查：每年1月1日凌晨01:00执行（Cron: `0 0 1 1 1 ?`）

   - 支持按任务类型、执行结果、时间范围筛选

### 定时任务配置说明

系统使用 Spring Boot 内置的 `@Scheduled` 定时任务框架，通过 Cron 表达式控制执行频率。

#### 定时任务配置位置

- **启动类**：`Application.java` 中已添加 `@EnableScheduling` 注解启用定时任务
- **任务类**：`task/MemberLevelScheduledTask.java`

#### 定时任务列表

| 任务名称 | 任务描述 | Cron 表达式 | 执行频率 |
|----------|----------|-------------|----------|
| 升级检查 | 检查所有会员的升级条件，自动升级到符合条件的最高等级 | `0 0 1 * * ?` | 每日凌晨 01:00 |
| 保级降级检查 | 检查所有非最低等级会员的保级条件，未达标自动降级 | `0 0 1 1 1 ?` | 每年 1月1日 凌晨 01:00 |
| 年度消费重置 | 将所有会员的本年度消费金额重置为0 | 在保级降级检查后立即执行 | 每年 1月1日 |

#### 修改执行频率

如需修改定时任务执行频率，编辑 `backend/src/main/java/com/example/permission/task/MemberLevelScheduledTask.java` 文件中对应方法的 `@Scheduled(cron = "...")` 注解值。

常用 Cron 表达式示例：
```
0 0 1 * * ?        - 每日凌晨 1 点
0 0 2 * * ?        - 每日凌晨 2 点
0 0 0 1 * ?        - 每月 1 号凌晨 0 点
0 0 1 1 1 ?        - 每年 1月1日 凌晨 1 点
0 */30 * * * ?     - 每 30 分钟执行一次
```

#### 手动触发

除定时自动执行外，酒店管理员可通过前端页面（会员管理 → 定时任务管理）的"手动执行"按钮立即触发任务执行，用于测试或特殊场景。

2. **查看权重总和**：页面顶部显示"当前权重总和：XX%"，不等于100%时显示黄色警告提示
3. **新增指标**：点击"新增指标"按钮 → 填写指标名称（如"客房卫生"）、指标说明（选填）、权重百分比（0-100）、是否必评、显示排序、启用状态 → 提交保存
4. **编辑指标**：点击列表行的"编辑"按钮 → 修改指标信息 → 保存
5. **删除指标**：点击"删除"按钮 → 确认后删除（至少保留一个启用指标）
6. **启用/禁用指标**：点击状态开关快速切换指标启用状态
7. **预览评价页面**：点击"预览"按钮 → 弹窗展示星级评分控件、必评指标红星标记、综合评分计算说明

#### 评价标签与快捷评语管理
1. **进入页面**：评价管理 → 评价标签与评语
2. **好评标签管理**：
   - 切换到"好评标签"标签页
   - 点击"添加标签" → 输入标签文本（2-10字，如"房间宽敞""服务周到"）→ 保存
   - 点击标签上的"编辑"修改文本，"上移/下移"调整顺序，"删除"移除标签
3. **差评标签管理**：
   - 切换到"差评标签"标签页
   - 操作同好评标签，预置如"房间老旧""卫生不佳"等
4. **快捷评语管理**：
   - 切换到"快捷评语"标签页
   - 点击"添加评语" → 输入评语内容（5-100字）、选择分类（好评/中性/差评）→ 保存
   - 可编辑和删除评语
5. **标签预览**：点击"预览"按钮 → 弹窗展示好评/差评分组标签卡片，可模拟点击选中效果

### 评价系统-客户评价提交使用指南

#### 评价邀请管理
1. **进入页面**：使用 admin 或 hotel_admin 登录 → 评价管理 → 评价邀请管理
2. **查看邀请列表**：显示所有评价邀请记录，包含入住单号、客人信息、房型、入住日期、邀请时间、评价状态、发送状态等
3. **搜索筛选**：
   - 关键词搜索：输入客人姓名、手机号或入住单号搜索
   - 评价状态筛选：选择"待评价"或"已评价"
   - 日期范围筛选：选择邀请时间的起止日期
4. **复制评价链接**：点击"复制链接"按钮，完整评价链接自动复制到剪贴板，提示"链接已复制，可用于测试"
5. **发送评价邀请**：
   - 点击"发送邀请"按钮（仅未发送的记录显示）
   - 弹出对话框选择发送方式：短信发送 / 邮件发送
   - 确认后更新发送状态为"已发送"，记录发送时间和发送方式
   - 系统提示"评价邀请已发送（模拟）"，不实际发送消息
6. **查看评价详情**：点击"查看评价"按钮（仅已评价的记录显示），弹出对话框显示评价详情：
   - 综合评分
   - 各指标评分（星级显示）
   - 选中的评价标签
   - 评价内容
   - 评价图片（如有）

#### 手动创建评价邀请（入住单详情页）
1. **进入入住单详情**：入住管理 → 入住单管理 → 点击某条已退房记录的"详情"
2. **点击"邀请评价"按钮**（仅已退房的入住单显示）
3. **系统判断逻辑**：
   - 未退房：提示"客人未退房，无法邀请评价"
   - 已有评价邀请：提示"已创建评价邀请"，并显示现有评价链接
   - 已退房且未邀请：显示创建确认对话框
4. **创建成功**：显示评价链接，可点击"复制链接"按钮复制

#### 评价链接测试方法
1. **获取测试链接**：
   - 方式一：评价邀请管理 → 点击"复制链接"获取
   - 方式二：入住单详情 → 邀请评价 → 创建后复制链接
   - 链接格式示例：`http://localhost:8080/h5/review?order=CI202606150001&code=a1b2c3d4`
2. **访问H5评价页面**：
   - 直接在浏览器中粘贴链接访问
   - 或在评价邀请管理页点击"复制链接"后，打开新标签页粘贴访问
3. **测试不同场景**：
   - **正常访问**：使用有效的未过期、未评价的链接，可正常进入评价页面
   - **链接已过期**：使用超过7天有效期的链接，显示"链接已失效"
   - **已评价**：使用已评价的链接，显示"您已评价过该订单"
   - **验证码错误**：修改链接中的code参数，显示"链接已失效"
4. **测试评价流程**：
   - 对必评指标进行星级评分（点击星星选择1-5分）
   - 查看综合评分实时更新
   - 点击选择评价标签（好评绿色、差评红色，可多选）
   - 点击快捷评语按钮，自动填充到文本域
   - 填写评价内容（最多500字，显示字数统计）
   - 上传图片（点击选择或拖拽，最多6张，支持JPG/PNG，单张≤5MB）
   - 勾选"匿名评价"选项
   - 点击底部"提交评价"按钮
   - 验证必评指标未评分时的提示
   - 提交成功后查看感谢页面，会员账号会显示积分赠送提示

#### 自动创建评价邀请（退房触发）
1. **办理退房**：入住管理 → 退房管理 → 选择入住单办理退房
2. **系统自动触发**：退房完成后，系统自动创建评价邀请记录
3. **验证结果**：评价邀请管理列表中可看到新创建的邀请记录，评价状态为"待评价"

#### 会员评价积分奖励测试

3. **提交评价**：使用该入住单的评价链接提交评价
4. **查看积分**：会员详情页 → 积分明细，可看到"评价积分"记录
5. **提交成功页**：显示"已赠送XX积分到您的账户"
